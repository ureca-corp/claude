# git-split-pr

세션에서 변경된 파일들을 업무 단위별로 자동 분류하고, 각 단위마다 독립 브랜치 생성 → 한글 커밋 → 푸시 → PR 생성 → 리뷰 모니터링까지 완전한 Git 워크플로우를 자동화하는 스킬.

## 주요 기능

- **변경사항 분석** — `git diff`로 세션 전체 변경 파악
- **업무 단위 분류** — 파일을 기능/유형별로 논리적 그룹화 후 사용자 확인
- **브랜치 자동 생성** — 각 단위별 `feat/`, `fix/`, `refactor/` 브랜치 생성
- **한글 커밋 메시지** — TLDR 제목 + 기능 관점 리스트 형식
- **PR 자동 생성** — `gh pr create`로 main 브랜치 대상 PR 등록
- **리뷰 모니터링** — 리뷰 항목 있으면 `/loop`으로 1분마다 상태 추적

## 트리거 예시

- "diff 확인하고 PR 나눠줘"
- "변경사항 브랜치로 분리해줘"
- "한글 커밋으로 PR 올려줘"
- "업무 단위별 브랜치 만들어"
- "PR 리뷰 모니터링해줘"

## 커밋 메시지 형식

```
feat: 소셜 계정으로 로그인할 수 있는 기능 추가

- 구글, 카카오 계정으로 로그인 가능
- 기존 이메일 계정과 소셜 계정 연동 지원
- 로그인 화면에 소셜 로그인 버튼 추가
```

## 요구사항

- `git` CLI
- `gh` CLI (GitHub CLI, 인증 완료 상태)
- `/loop` 스킬 (PR 모니터링 시 사용)
