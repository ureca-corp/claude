```markdown
# {Domain} 기능 정의

## 📋 주요 기능

### 1. {기능 이름}

**사용자 시나리오**:
\`\`\`
사용자: {상황}
1. {액션 1}
2. {액션 2}
3. 결과: {결과}
\`\`\`

- {입력 1}
- {입력 2}

**출력**:
- {출력 1}
- {출력 2}

- {제외 사항 1}
- {제외 사항 2}
```

## 주요 기능 작성

### 기능 이름

api-spec.md의 API 이름 또는 주요 액션:

**예시**:
- 회원가입
- 프로필 조회
- 프로필 수정
- 회원 탈퇴
- 번역 요청
- 번역 기록 조회
- 미션 시작
- 미션 진행 상황 확인

---

### 설명 (Description)

**템플릿**:
```markdown
**설명**: {이 기능이 무엇을 하는지 한 줄로}
```

**작성 원칙**:
- 한 줄로 요약
- "~한다" 형태
- 사용자 관점

**예시**:
```markdown
**설명**: 새로운 사용자 계정을 생성한다
**설명**: 사용자의 번역 기록 전체를 조회한다
**설명**: 현재 진행 중인 미션의 완료 비율을 확인한다
```

---

### 사용자 시나리오 (User Scenario)

**템플릿**:
```markdown
**사용자 시나리오**:
\`\`\`
사용자: {어떤 상황에서}
1. {첫 번째 액션}
2. {두 번째 액션}
3. {세 번째 액션}
4. 결과: {최종 결과}
\`\`\`
```

**작성 원칙**:
- 구체적인 사용자 페르소나 (예: "여행자", "외국인 방문객")
- 3-5단계의 순차적 액션
- 마지막은 "결과:"로 시작

**예시 1: 회원가입**
```markdown
**사용자 시나리오**:
\`\`\`
여행자: 앱을 처음 사용한다
1. 이메일과 닉네임을 입력한다
2. 선호 언어를 선택한다 (선택사항)
3. 프로필 사진을 업로드한다 (선택사항)
4. 결과: 계정이 생성되고 자동 로그인된다
\`\`\`
```

**예시 2: 번역 요청**
```markdown
**사용자 시나리오**:
\`\`\`
외국인 방문객: 식당에서 메뉴를 보고 있다
1. 번역할 텍스트를 입력한다 (예: "삼겹살")
2. 목표 언어를 선택한다 (예: 영어)
3. 번역 버튼을 누른다
4. 결과: "Pork belly"가 표시되고 번역 기록에 저장된다
\`\`\`
```

**예시 3: 미션 시작**
```markdown
**사용자 시나리오**:
\`\`\`
학습자: 한국어 실력을 향상시키고 싶다
1. 미션 목록에서 "택시 타기" 미션을 선택한다
2. 미션 설명과 목표를 확인한다
3. "시작하기" 버튼을 누른다
4. 결과: 미션이 시작되고 진행 상황이 0%로 표시된다
\`\`\`
```

---

### 입력 (Input)

**템플릿**:
```markdown
**입력**:
- {필드명} ({필수/선택}) - {설명}
- {필드명} ({필수/선택}) - {설명}
```

**작성 원칙**:
- api-spec.md의 Request 필드 기반
- 필수/선택 명시
- 간단한 설명만

**예시**:
```markdown
**입력**:
- 이메일 (필수) - 로그인용 이메일 주소
- 닉네임 (필수) - 사용자 표시 이름
- 프로필 사진 (선택) - 프로필 이미지 URL
- 선호 언어 (선택) - 기본 언어 설정 (한국어/영어)
```

---

### 출력 (Output)

**템플릿**:
```markdown
**출력**:
- {항목 1}
- {항목 2}
```

**작성 원칙**:
- api-spec.md의 Response 필드 기반
- 주요 항목만 나열
- 사용자 관점에서 의미 있는 것만

**예시 1: 회원가입**
```markdown
**출력**:
- 사용자 ID
- 가입 시각
- 프로필 정보 (이메일, 닉네임, 사진, 언어)
```

**예시 2: 번역 요청**
```markdown
**출력**:
- 번역 결과 텍스트
- 번역 기록 ID
- 생성 시각
```

**예시 3: 미션 진행 상황 확인**
```markdown
**출력**:
- 미션 이름
- 현재 진행률 (%)
- 완료한 번역 개수
- 필요한 번역 개수
- 상태 (진행 중/완료)
```

## 범위 밖 (Out of Scope)

**템플릿**:
```markdown
## 🚫 범위 밖 (Out of Scope)

- {제외 사항 1}
- {제외 사항 2}
- {제외 사항 3}
```

**작성 원칙**:
- MVP에서 제외된 기능
- 비슷하지만 하지 않는 것
- 향후 고려 사항

**예시 (users 도메인)**:
```markdown
## 🚫 범위 밖 (Out of Scope)

- 소셜 로그인 (Google, Apple 등)
- 이메일 인증
- 비밀번호 찾기
- 관리자 권한 관리
- 사용자 등급/레벨 시스템
```

**예시 (translations 도메인)**:
```markdown
## 🚫 범위 밖 (Out of Scope)

- 음성 번역 (TTS/STT)
- 사진 속 텍스트 번역 (OCR)
- 실시간 대화 번역
- 번역 품질 평가
- 다른 사용자와 번역 공유
```

**예시 (missions 도메인)**:
```markdown
## 🚫 범위 밖 (Out of Scope)

- 미션 생성 기능 (관리자만 가능)
- 미션 난이도 조절
- 팀 미션 (여러 명이 함께)
- 미션 보상/포인트 시스템
- 미션 리더보드
```

## 완전한 예시

### users 도메인

```markdown
# users 기능 정의

## 📋 주요 기능

### 1. 회원가입

**설명**: 새로운 사용자 계정을 생성한다

**사용자 시나리오**:
\`\`\`
여행자: 앱을 처음 사용한다
1. 이메일과 닉네임을 입력한다
2. 선호 언어를 선택한다 (선택사항)
3. 프로필 사진을 업로드한다 (선택사항)
4. 결과: 계정이 생성되고 자동 로그인된다
\`\`\`

**입력**:
- 이메일 (필수) - 로그인용 이메일 주소
- 닉네임 (필수) - 사용자 표시 이름
- 프로필 사진 (선택) - 프로필 이미지 URL
- 선호 언어 (선택) - 기본 언어 설정 (한국어/영어)

**출력**:
- 사용자 ID
- 가입 시각
- 프로필 정보 (이메일, 닉네임, 사진, 언어)

---

### 2. 프로필 조회

**설명**: 자신의 프로필 정보를 조회한다

**사용자 시나리오**:
\`\`\`
사용자: 내 정보를 확인하고 싶다
1. 프로필 화면에 접속한다
2. 결과: 이메일, 닉네임, 프로필 사진, 선호 언어가 표시된다
\`\`\`

**입력**:
- 사용자 ID (필수) - 조회할 사용자 식별자

**출력**:
- 사용자 ID
- 이메일
- 닉네임
- 프로필 사진
- 선호 언어
- 가입 시각

---

### 3. 프로필 수정

**설명**: 자신의 프로필 정보를 변경한다

**사용자 시나리오**:
\`\`\`
사용자: 닉네임을 바꾸고 싶다
1. 프로필 편집 화면에 접속한다
2. 닉네임을 새로운 이름으로 변경한다
3. 저장 버튼을 누른다
4. 결과: 프로필이 업데이트되고 새 닉네임이 표시된다
\`\`\`

**입력**:
- 사용자 ID (필수) - 수정할 사용자 식별자
- 닉네임 (선택) - 새로운 표시 이름
- 프로필 사진 (선택) - 새로운 이미지 URL
- 선호 언어 (선택) - 새로운 언어 설정

**출력**:
- 업데이트된 프로필 정보

---

### 4. 회원 탈퇴

**설명**: 계정과 모든 관련 데이터를 삭제한다

**사용자 시나리오**:
\`\`\`
사용자: 앱을 더 이상 사용하지 않기로 결정했다
1. 설정 화면에서 "탈퇴하기"를 선택한다
2. 경고 메시지를 확인한다
3. 탈퇴를 최종 확인한다
4. 결과: 계정과 모든 번역 기록, 미션 진행 기록이 삭제된다
\`\`\`

**입력**:
- 사용자 ID (필수) - 삭제할 사용자 식별자

**출력**:
- 완료 메시지
- 삭제된 데이터 개수

---

## 🚫 범위 밖 (Out of Scope)

- 소셜 로그인 (Google, Apple 등)
- 이메일 인증
- 비밀번호 찾기
- 관리자 권한 관리
- 사용자 등급/레벨 시스템
```
