### 설명 (Description)

**템플릿**:
```markdown
**설명**: {이 기능이 무엇을 하는지 한 줄로}
```

**작성 원칙**:
- 한 줄로 요약
- "~한다" 형태
- 사용자 관점

**예시**:
```markdown
**설명**: 새로운 사용자 계정을 생성한다
**설명**: 사용자의 번역 기록 전체를 조회한다
**설명**: 현재 진행 중인 미션의 완료 비율을 확인한다
```

---

### 사용자 시나리오 (User Scenario)

**템플릿**:
```markdown
**사용자 시나리오**:
\`\`\`
사용자: {어떤 상황에서}
1. {첫 번째 액션}
2. {두 번째 액션}
3. {세 번째 액션}
4. 결과: {최종 결과}
\`\`\`
```

**작성 원칙**:
- 구체적인 사용자 페르소나 (예: "여행자", "외국인 방문객")
- 3-5단계의 순차적 액션
- 마지막은 "결과:"로 시작

**예시 1: 회원가입**
```markdown
**사용자 시나리오**:
\`\`\`
여행자: 앱을 처음 사용한다
1. 이메일과 닉네임을 입력한다
2. 선호 언어를 선택한다 (선택사항)
3. 프로필 사진을 업로드한다 (선택사항)
4. 결과: 계정이 생성되고 자동 로그인된다
\`\`\`
```

**예시 2: 번역 요청**
```markdown
**사용자 시나리오**:
\`\`\`
외국인 방문객: 식당에서 메뉴를 보고 있다
1. 번역할 텍스트를 입력한다 (예: "삼겹살")
2. 목표 언어를 선택한다 (예: 영어)
3. 번역 버튼을 누른다
4. 결과: "Pork belly"가 표시되고 번역 기록에 저장된다
\`\`\`
```

**예시 3: 미션 시작**
```markdown
**사용자 시나리오**:
\`\`\`
학습자: 한국어 실력을 향상시키고 싶다
1. 미션 목록에서 "택시 타기" 미션을 선택한다
2. 미션 설명과 목표를 확인한다
3. "시작하기" 버튼을 누른다
4. 결과: 미션이 시작되고 진행 상황이 0%로 표시된다
\`\`\`
```

### 입력 (Input)


**작성 원칙**:
- api-spec.md의 Request 필드 기반
- 필수/선택 명시
- 간단한 설명만

**예시**:
```markdown
**입력**:
- 이메일 (필수) - 로그인용 이메일 주소

- 선호 언어 (선택) - 기본 언어 설정 (한국어/영어)
```

## 📱 화면 구성 작성

### 화면 설계 원칙

> 1억 명 이상이 사용하는 서비스(Instagram, Facebook, Twitter, YouTube, KakaoTalk 등)의 검증된 UX 패턴을 참고한다. 단, 그대로 복제하지 않고 **심플하고 미니멀한 방향**으로 재해석한다.

**핵심 관점** (`frontend-design` 스킬 연동):
- **정보 계층 (Information Hierarchy)**: 가장 중요한 정보가 시선을 먼저 사로잡는가?
- **인터랙션 플로우 (Interaction Flow)**: 사용자의 다음 행동이 자연스러운가?
- **감성 디자인 (Emotional Design)**: 사용자가 이 화면에서 어떤 감정을 느끼는가?
- **접근성 (Accessibility)**: 다양한 사용자가 불편 없이 사용할 수 있는가?
- **컨텍스트 (Context)**: 사용자가 이 화면에 도달하는 상황은?

**안티패턴**:
- ❌ 단순 CRUD 나열 (목록 → 상세 → 수정 → 삭제)
- ❌ 데이터베이스 필드를 그대로 화면에 배치
- ❌ 기능 중심 설계 (기능이 아닌 사용자 목표 중심으로)
- ❌ 과도한 정보 표시 (한 화면에 너무 많은 정보)

**좋은 패턴**:
- ✅ 사용자 목표 중심 설계 (인스타그램: "순간을 공유한다" → 카메라 버튼이 중앙)
- ✅ 점진적 정보 공개 (페이스북: 피드에서 요약 → 탭하면 상세)
- ✅ 빈 상태(Empty State) 설계 (첫 사용자가 보는 화면)
- ✅ 로딩/에러/성공 상태 고려

### 템플릿

```markdown

- **경로**: /{path}
- **도메인**: {domain}
- **화면 목적**: {사용자 관점에서 이 화면의 핵심 가치 한 줄}
- **UX 레퍼런스**: {참고할 대형 서비스의 유사 화면}
- **핵심 인터랙션**:
  - {사용자의 주요 행동 1}
  - {사용자의 주요 행동 2}
- **정보 구조**: {화면에 표시되는 정보의 우선순위 - 가장 중요한 것부터}
- **네비게이션**: {성공 → /path, 취소 → /path}
- **관련 기능**: 기능 {N}
```

```markdown
## 📱 화면 구성

> **디자인 원칙**: Instagram, KakaoTalk 등 1억 명 이상 사용하는 서비스의 UX를 참고하되, 심플하고 미니멀한 구성을 지향한다. `frontend-design` 스킬을 활용하여 디자인 품질을 확보한다.

### 화면 1: 로그인

- **경로**: /login
- **도메인**: users
- **화면 목적**: 최소한의 마찰로 앱에 진입한다
- **UX 레퍼런스**: Instagram 로그인 (로고 + 단 2개 입력필드 + 소셜 로그인)
- **핵심 인터랙션**:
  - 이메일/비밀번호 입력 후 로그인
  - 소셜 로그인 원탭 진입
- **정보 구조**: 브랜드 로고 > 입력 폼 > 소셜 로그인 > 회원가입 링크
- **네비게이션**: 성공 → /home, 회원가입 → /register
- **관련 기능**: 기능 1 (회원가입)

---

### 화면 2: 프로필

- **경로**: /profile
- **도메인**: users
- **화면 목적**: 내 정보를 한눈에 확인하고 편집한다
- **UX 레퍼런스**: Instagram 프로필 (프로필 사진 + 핵심 숫자 + 편집 버튼)
- **핵심 인터랙션**:
  - 프로필 사진/닉네임 확인
  - "편집" 탭하여 수정 모드 진입
- **정보 구조**: 프로필 사진 > 닉네임 > 활동 요약 > 설정
- **네비게이션**: 편집 → /profile/edit, 설정 → /settings
- **관련 기능**: 기능 2 (프로필 조회), 기능 3 (프로필 수정)
```
