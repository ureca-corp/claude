3. [API 명세](./api-spec.md) - API 상세 설계
4. [비즈니스 규칙](./business-rules.md) - 제약조건, 정책

---

## 📊 요약

- **API 수**: {수}개

## 역할 (Role) 작성

### 템플릿

```markdown
> **역할**: {도메인의 주요 책임을 한 줄로}
```

### 예시

| 도메인 | 역할 |
|--------|------|
| users | 앱 사용자 관리 |
| translations | 텍스트 번역 및 기록 관리 |
| missions | 미션 진행 상태 추적 |
| phrases | 자주 쓰는 표현 모음 관리 |
| orders | 주문 생성 및 결제 처리 |
| products | 상품 정보 관리 |

---

## 도메인 관계 (Domain Relationships)

### 독립 도메인 (의존 없음)

```markdown
## 🔗 도메인 관계

### 의존하는 도메인
- 없음 (독립 도메인)

### 이 도메인을 사용하는 도메인
- `translations` - 번역 기록 소유자 확인
- `missions` - 미션 진행자 확인
```

### 의존 도메인 (다른 도메인 참조)

```markdown
## 🔗 도메인 관계

### 의존하는 도메인
- `users` - 번역 요청자 정보 필요
- `phrases` - 추천 표현 조회

### 이 도메인을 사용하는 도메인
- `missions` - 번역 완료 개수 조회
```

### 설명 작성 원칙

- **의존하는 도메인**: 왜 그 도메인이 필요한지 설명
- **사용하는 도메인**: 어떻게 이 도메인을 활용하는지 설명

**예시**:
```markdown
### 의존하는 도메인
- `users` - 번역 요청자 정보 필요
- `phrases` - 추천 표현 조회

### 이 도메인을 사용하는 도메인
- `missions` - 번역 완료 개수 조회
```

## 요약 (Summary)

### 템플릿

```markdown
## 📊 요약

- **엔티티 수**: {수}개 ({엔티티 이름 나열})
- **API 수**: {수}개
- **상태 전이**: {있음/없음}
```

### 엔티티 수 계산

domain-model.md에서 정의된 엔티티 추출:

**예시 (users 도메인)**:
```markdown
- **엔티티 수**: 1개 (User)
```

**예시 (orders 도메인)**:
```markdown
- **엔티티 수**: 2개 (Order, OrderItem)
```

### API 수 계산

api-spec.md의 API 목록에서 개수 확인:

**예시**:
```markdown
- **API 수**: 4개
```

### 상태 전이 확인

domain-model.md 또는 business-rules.md에서 FSM 존재 여부:

**예시 (상태 전이 없음)**:
```markdown
- **상태 전이**: 없음
```

**예시 (상태 전이 있음)**:
```markdown
- **상태 전이**: 있음 (NOT_STARTED → IN_PROGRESS → COMPLETED)
```

## 핵심 가치 (Core Value)

### 템플릿

```markdown
## 🎯 핵심 가치

{이 도메인이 시스템에 제공하는 핵심 가치를 한 문장으로}
```

### 작성 원칙

- **한 문장**으로 요약
- **무엇을 하는가** + **왜 중요한가**
- 기술 용어 사용 금지
- 비즈니스 가치 중심

### 예시

| 도메인 | 핵심 가치 |
|--------|-----------|
| users | 사용자 정보를 안전하게 관리하고 다른 도메인에 신원 정보를 제공한다 |
| translations | 다국어 번역 기능을 제공하여 언어 장벽을 해소한다 |
| missions | 학습 목표를 설정하고 진행 상황을 추적하여 사용자 참여를 유도한다 |
| phrases | 자주 쓰는 표현을 빠르게 찾을 수 있도록 하여 번역 효율을 높인다 |
| orders | 주문 생성부터 결제까지의 흐름을 관리하여 안전한 거래를 보장한다 |
| products | 상품 정보를 체계적으로 관리하여 정확한 주문을 가능하게 한다 |
